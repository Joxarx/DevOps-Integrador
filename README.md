## Descripción del proyecto

**SmartCampus Services** es una aplicación web basada en microservicios diseñada para gestionar servicios universitarios como:

- Reservación de aulas
- Gestión de eventos académicos
- Consulta de horarios
- Notificaciones institucionales

El sistema está pensado para ser escalable, mantenible y altamente disponible, lo que lo convierte en un candidato ideal para la implementación de la metodología **DevOps**.

---

## Objetivo del proyecto

Aplicar la metodología DevOps para mejorar el ciclo de vida del desarrollo de software mediante la automatización, integración continua, entrega continua y colaboración entre los equipos de desarrollo y operaciones.

---

## Metodología DevOps

La metodología DevOps se implementa en este proyecto con el fin de:

- Reducir tiempos de entrega
- Minimizar errores humanos
- Mejorar la calidad del software
- Facilitar despliegues frecuentes y confiables

DevOps permite integrar prácticas de desarrollo (Dev) y operaciones (Ops) en un flujo continuo apoyado por herramientas de automatización.

## Estrategia de ramificación

Se eligió una estrategia basada en **Git Flow**, la cual facilita el trabajo colaborativo y la integración continua.

### Ramas utilizadas
- `main`: versión estable del sistema
- `develop`: rama de desarrollo
- `feature/*`: desarrollo de nuevas funcionalidades
- `hotfix/*`: corrección de errores críticos

Esta estrategia permite mantener un código estable y controlar los cambios de forma organizada.
